# MPU6050 Module

## Overview

- This module provides a blocking MPU6050 driver built on top of `drvAnlogIic`.
- The current hardware mapping uses `DRVANLOGIIC_BUS0` on `PD12` as `SCL` and `PD13` as `SDA`.
- The default device address is `0x68`. Use `0x69` when the MPU6050 `AD0` pin is pulled high.

## Files

- `mpu6050.h`: Public API, status codes, configuration structure, and raw sample type.
- `mpu6050.c`: Device initialization, register access helpers, sleep control, and raw sample reads.

## Initialization Flow

1. Call `mpu6050GetDefaultConfig()`.
2. Adjust `address`, `sampleRateDivider`, `dlpfConfig`, `accelRange`, or `gyroRange` if needed.
3. Call `mpu6050Init()` once before any read or write operation.

## Public API

- `mpu6050Init()`: Initializes the software IIC bus, verifies `WHO_AM_I`, resets the device, and applies the selected configuration.
- `mpu6050ReadRawSample()`: Reads one 14-byte sensor frame containing accel, temperature, and gyro raw values.
- `mpu6050ReadTemperatureCentiDegC()`: Converts the raw temperature sample to centi-degrees Celsius.
- `mpu6050SetSleepEnabled()`: Enables or disables the sleep bit in `PWR_MGMT_1`.
- `mpu6050ReadRegister()` and `mpu6050WriteRegister()`: Expose direct single-register access for additional tuning.

## Verification Notes

- Build verification confirms that the module and port binding compile into the current firmware.
- Hardware verification is still required to confirm pull-up sizing, signal integrity, and the expected `WHO_AM_I` value on the target board.
